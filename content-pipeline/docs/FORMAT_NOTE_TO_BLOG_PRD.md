# 部落格貼文生成規格書 (Blog Post PRD)

## 概述
此規格書定義了基於提供素材生成部落格貼文的標準流程和輸出格式。

## 輸入素材
- 網頁內容
- 文字段落
- 其他相關資料

## 輸出格式

### 1. Article ID
- **格式**: SEO 最佳化的文章 ID
- **規則**: 
  - 使用小寫字母和連字符
  - 包含關鍵字
  - 長度控制在 50 字以內
  - 範例: `ai-coding-assistant-best-practices-2024`

### 2. Title
- **格式**: SEO 最佳化的文章標題
- **規則**:
  - 包含主要關鍵字
  - 長度控制在 60 字以內
  - 吸引點擊且具描述性
  - 範例: `AI 程式設計助手最佳實踐：提升開發效率的 10 個技巧`

### 3. SubTitle
- **格式**: SEO 最佳化的文章副標題
- **規則**:
  - 補充說明主標題
  - 包含次要關鍵字
  - 長度控制在 100 字以內
  - 範例: `從基礎設定到進階應用，完整解析如何善用 AI 工具優化程式開發流程`

### 4. Description
- **格式**: SEO 最佳化的文章描述
- **規則**:
  - 約 50 字
  - 包含主要關鍵字
  - 吸引讀者點擊
  - 準確描述文章內容
  - 範例: `探索 AI 程式設計助手的最佳使用方式，學習提升開發效率的實用技巧與策略。`

### 5. Content
- **格式**: SEO 最佳化的完整文章內容
- **規則**:
  - 基於提供素材撰寫
  - 使用 `/polish 90%` 進行文字潤稿優化
  - 結構清晰：引言、主體、結論
  - 素材可能沒有結論，若沒有結論，請自行補充結論，約 300 字左右。
  - 包含適當的標題層級 (H2, H3)
  - 長度建議：1000-3000 字
  - 包含相關關鍵字但避免過度堆砌

## 文字優化標準

### Polish 指令規則
- **預設**: 保持原文 80% 字數
- **自訂**: 可指定百分比 (如 90%)
- **限制**: 不得修改或刪除 `\`...\`` 程式碼區塊內容
- **格式**: 避免使用列表格式，以段落形式呈現
- **範圍**: 僅針對選擇部分進行修改

### 文字品質要求
- 語句通順流暢
- 用詞精確專業
- 邏輯結構清晰
- 符合 SEO 最佳實踐
- 保持原文核心訊息

## 使用流程

1. **提供素材**: 輸入網頁內容或文字段落
2. **分析內容**: 提取關鍵資訊和主題
3. **生成結構**: 規劃文章架構
4. **撰寫內容**: 基於素材創作文章
5. **優化潤稿**: 使用 polish 指令精修文字
6. **輸出格式**: 按照規格書格式整理

## 範例輸出

```markdown
# Article ID
ai-coding-assistant-best-practices-2024

# Title
AI 程式設計助手最佳實踐：提升開發效率的 10 個技巧

# SubTitle
從基礎設定到進階應用，完整解析如何善用 AI 工具優化程式開發流程

# Content
[基於素材撰寫的完整文章內容，經過 90% polish 優化]

# Description
探索 AI 程式設計助手的最佳使用方式，學習提升開發效率的實用技巧與策略。
```

## 注意事項

- 確保所有輸出內容符合 SEO 最佳實踐
- 保持內容的原創性和價值性
- 適當引用來源並標註參考資料
- 定期更新關鍵字策略以符合搜尋趨勢
- 測試不同標題和描述的點擊率表現
